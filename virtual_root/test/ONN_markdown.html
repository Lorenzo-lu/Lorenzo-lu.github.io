    We investigated a feasibility of using a neural network to optimize physical masks in single-pixel imaging under Poisson noise. In this document, the important notes and fundamental principles are listed for reference.

    Single-pixel imaging

    Single-pixel imaging uses a physical mask to measure the sum of photons from different parts of the image x\mathbf{x}x. After adequate measurements (may fewer than the number of pixels), it is possible to reconstruct the image. Mathematically,

    y1×m=x1×nMn×m\mathbf{y}_{1\times m} = \mathbf{x}_{1\times n} \mathbf{M}_{n\times m} y1×m​=x1×n​Mn×m​

    Also, in the cases where reconstruction is not necessarily required, we can use the mask to get the similarity vector as the input to a software neural network. If we can extend the network to the physical mask and optimize it via backpropagation, we call it an Optical Neural Network.

    Backpropagation under Poisson noise

    Suppose we have a mask basis M\mathbf{M}M, and the field of view is notated by x\mathbf{x}x. The measurement y~∼P(y)=P(xM)\mathbf{\tilde{y}} \sim \mathcal{P}(\mathbf{y}) = \mathcal{P}(\mathbf{x}\mathbf{M})y~​∼P(y)=P(xM). Since the optical neural network consists of two parts, the DMD hardware part and the software part, the later part is essentially optimized under noise, which is unavoidable. The question is how can we optimize the M\mathbf{M}M in a correct and fast way.

    • If contain Poisson noise:
      Essentially, the noisy measurement y~\mathbf{\tilde{y}}y~​ is the input to the software part, and the kernel ∂l~∂y~k\frac{\partial \tilde{l}}{\partial \tilde{y}_k}∂y~​k​∂l~​ is the corresponding gradient from the software part. There is nothing we can do with it. The non-linearity introduced by the activation functions is one reason that we cannot simply let ∂yk∂y~k≡1\frac{\partial y_k}{\partial \tilde{y}_k} \equiv 1∂y~​k​∂yk​​≡1.

    One method is to treat the Poisson noise as an activation function. In each epoch, I assign a new weight based on the Poisson distribution on it. When backpropagate, the ration between the assigned value and the real value will be used as the coefficiencts. To explain mathematically, ∂l~k∂Mj,k=∂l~k∂ykxj=∂l~k∂y~kdy~kdykxj=∂l~k∂(yky~ky~k)xj=∂l~k∂ykxj\frac{\partial \tilde{l}_k}{\partial M_{j,k}} = \frac{\partial \tilde{l}_k}{\partial {y}_k} x_j = \frac{\partial \tilde{l}_k}{\partial \tilde{y}_k} \frac{d \tilde{y}_k}{d {y}_k} x_j = \frac{\partial \tilde{l}_k}{\partial \left(\frac{{y}_k}{\tilde{y}_k} \tilde{y}_k \right)} x_j = \frac{\partial \tilde{l}_k}{\partial {y}_k} x_j∂Mj,k​∂l~k​​=∂yk​∂l~k​​xj​=∂y~​k​∂l~k​​dyk​dy~​k​​xj​=∂(y~​k​yk​​y~​k​)∂l~k​​xj​=∂yk​∂l~k​​xj​. This holds the form of an activation function. But the drawback is , the loss itself is noisy. It is hard to predict how the l~\tilde{l}l~ changes versus the random noise, so this method may have some fluctuation. This method essentially apply another random mask (obeying a Poiss1×m\text{Poiss}_{1\times m}Poiss1×m​ ) on the measurement y\mathbf{y}y. The fluctuation arises from this random mask applied.

    The code:

    class Poisson(nn.Module):
    def __init__(self):
        super().__init__();
    def forward(self, x):           
        out = Variable(torch.poisson(x) / (x+1e-16)) * (x+1e-16);
        return out

    class Poisson(nn.Module):
    def __init__(self):
        super().__init__();
    def forward(self, x):           
        out = Variable(torch.poisson(x) - x) + x;
        return out

    Figure: Fair comparison

    They both holds the same architecture. Based on the figures and outputs, the former approach is slightly better. My hypothesis is the noise on the loss l~\tilde{l}l~ is smoothed inside the software part of the neural network, while the later method leaks the fluctuation outside the software part.

    We can think about this process from another point: the software part of the neural network has the correct gradient in each epoch. If we just use the later one, that means the noise will be taken into optimizing mask M\mathbf{M}M. Mathematically, ∂l~k∂Mj,k=∂l~k∂y~kxj\frac{\partial \tilde{l}_k}{\partial M_{j,k}} = \frac{\partial \tilde{l}_k}{\partial \tilde{y}_k} x_j∂Mj,k​∂l~k​​=∂y~​k​∂l~k​​xj​. In this case, ∂y~k∂Mj,k=xj\frac{\partial \tilde{y}_k}{\partial M_{j,k}} = x_j∂Mj,k​∂y~​k​​=xj​. So we can see it is optimizing M\mathbf{M}M under y~=xM~\mathbf{\tilde{y}} = \mathbf{x\tilde{M}}y~​=xM~ where the noise is included in M\mathbf{M}M. But if we use the first strategy, the loss will fluctuate versus the epochs because of the noise introduced, but during each epoch, the gradient ∂l~k∂Mj,k\frac{\partial \tilde{l}_k}{\partial M_{j,k}}∂Mj,k​∂l~k​​ can be ensured to be correct. That is the reason for choosing the ratio method (the first one).

    Prospection and implication

    • The analogy between the noise introduced in the DMD layer and an "noise" activation function shows the feasibility of avoiding overfitting and improving neural networks' robustness against noise in imaging applications.

    • The attempts of using multiple patterns to make approximate a physical mask provides a solution to the long-term mask imaging question in mask imaging methods.
