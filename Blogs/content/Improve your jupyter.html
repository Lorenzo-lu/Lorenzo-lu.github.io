Run other languages in Jupyter notebook!

Why Jupyter?

If there is a bug in the 90th row of your python codes containing 100 rows, most computations is wasted if you just type 'python xxx.py' in terminal. If you are using a Jupyter notebook, you may just run the last cell to see where the bug is. That means, it is very Efficient especially when your project is Huge.

But Jupyter is not only for Python!

Here I will list how to run other laguanges in Jupyter! (I am using a MacBook Pro M1 and a Linux PC, and I didn't try with Windows).

MATLAB

  1. In your terminal, get the MATLAB Kernel for Python. https://www.mathworks.com/help/matlab/matlab_external/install-the-matlab-engine-for-python.html
    Run pip install matlab_kernel. Please be care with your Python version. I have a Python3.9 on macOS, but it doesn't work with the following step. My solution is installing Python3.8 aside of 3.9.
  2. Go to your MATLAB path; if you don't know where it is, just open your MATLAB and run matlabroot in the command. Then in terminal cd <matlabroot>/extern/engine/python. Then python setup.py install. Currently this installation is only compatible with Python2.7 and Python3.5-3.8, so check your python version first.
  3. Then run jupyter-notebook and you will be able to create your code in matlab by Jupyter! Don't forget to add the path to the environment: export PATH='<Your Jupyter Path>:$PATH. To double-check if it is valid, type echo $PATH.

A great advantage from my point of view is, you can create .m files in the cell. In MATLAB software, I usually feel exhausted when I have to switch between windows to debug. That super small coding area is really a nightmare! But in Jupyter, I only have to create functions in different cells and can view them directly. You may say MATLAB itself also support notebook, or .mlx. That is correct. However, after went through several crashes when trying to open some big .mlx files, I think I won't use it any more. By the way, in matlab you can only write your functions in the bottom, which, to me, is really a bad design. In Jupyter, you can write your codes at very beginning and don't have to scroll up and down frequently~.

Mathematica

Mathematica is a clever software which can even tell you the expressions in Mathematical form. This powerful tool can also be embeded into Jupyter.

https://github.com/WolframResearch/WolframLanguageForJupyter

Clone this repo, and cd the dir, and run ./configure-jupyter.wls add . If this doesn't work, try ./configure-jupyter.wls add <Path to WolframKernel> <Path to Jupyter Kernel>. For example ./configure-jupyter.wls add "/usr/local/bin/WolframKernel" "/Home/User/Library/bin/Jupyter".

Julia

Julia is a new language for numerical computations.

  1. Install Julia
  2. Open Julia, try using Pkg
  3. Input Pkg.add("IJulia")

Then it should work.

Appendix:

IPython Kernel version 7.19.0 has a bug when hinting. You may have problem when TAB to auto-complete codes. You may either turn to previous version or try pip install -U jedi==0.17.2 parso==0.7.1 to fix it.
